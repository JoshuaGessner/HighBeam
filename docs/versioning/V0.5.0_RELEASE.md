## Release Summary

v0.5.0 focuses on **production-quality stability** and **reliable deployment infrastructure**. All core server operations (metrics, logging, graceful shutdown, TLS, deployment) are fully implemented. This release is suitable for community servers with 10-20 players.

**Key Theme:** "Stability Through Observability"

## Completed Features

### Server — Operational Stability

- ✅ **Graceful Shutdown** — SIGTERM/SIGINT signal handlers ensure clean disconnect and resource cleanup
- ✅ **TCP Keepalive** — 60-second idle timeout enforcement prevents zombie connections
- ✅ **Bandwidth Throttling** — Tick-rate-based UDP relay throttling limits network overhead
- ✅ **Memory Cleanup** — Stale rate limiter records pruned automatically
- ✅ **Connection Validation** — State machine protection against invalid transitions
- ✅ **Rate Limiting** — Configurable limits for auth attempts (5/60s), chat (10/10s), vehicle spawn (5/5s)

**Configuration:** [ServerConfig.toml](../../server/ServerConfig.toml)

### Server — Observability & Logging

- ✅ **Metrics Logging** — Periodic snapshots of player count, vehicle count, network packets, bandwidth, memory usage
- ✅ **Log Rotation** — Automated file rotation by size and cleanup by age
  - Configurable: `RotationMaxSizeMb` (default 100MB) and `RotationMaxDays` (default 7 days)
  - Rotated logs are timestamped (e.g., `server.log.20260330-153045.log`)
  - Automatic cleanup of archives older than retention period
- ✅ **Metrics Interval** — Configurable logging period (default 60 seconds)
  - Set to 0 to disable metrics logging
- ✅ **Memory Monitoring** — Per-process RSS memory tracking via sysinfo
- ✅ **Structured Logging** — Configurable log levels (trace, debug, info, warn, error)
- ✅ **Chat Logging** — Optional chat message archival when `LogChat=true`

**Configuration:**
```toml
[Logging]
Level = "info"
LogFile = "server.log"
LogChat = true
MetricsIntervalSec = 60
RotationMaxSizeMb = 100
RotationMaxDays = 7
```

**Example Metrics Output:**
```
player_count=12 vehicle_count=28 tcp_rx_packets=1204 tcp_tx_packets=1089 
udp_rx_packets=24000 udp_tx_packets=24012 mod_sync_packets=0 mod_sync_bytes=0 
memory_rss_mib=87.4 interval_sec=60 "Runtime metrics"
```

### Server — Security & TLS

- ✅ **TLS Support** — Optional encryption for TCP connections
  - Loads PEM-format certificates and private keys
  - Configuration-based enable/disable
  - Supports both custom certs and auto-generation placeholder (not yet implemented)
- ✅ **Input Validation** — All user inputs validated (usernames, passwords, chat, vehicle IDs)
- ✅ **Session Tokens** — 64-byte entropy + timestamp prevents collisions

**Configuration:**
```toml
[TLS]
Enabled = false
CertPath = "certs/server.pem"
KeyPath = "certs/key.pem"
AutoGenerate = false
```

**To Enable TLS:**
1. Generate certificates: `openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes`
2. Set `Enabled = true` in ServerConfig.toml
3. Set correct `CertPath` and `KeyPath`
4. Restart server

### Server — Configuration & Deployment

- ✅ **CLI Arguments:**
  - `--config <path>` — Custom config file location
  - `--headless` — Disables future GUI features (for Docker/systemd)
  - `--help` — Show help message
- ✅ **Config Validation** — Entire config validated on startup; rejects invalids
- ✅ **Configurable Tick Rate** — Network update frequency (default 20 Hz)
- ✅ **Systemd Service** — Ready-to-use service template: [highbeam-server.service](../../server/highbeam-server.service)
- ✅ **Docker Support** — Dockerfile and docker-compose for containerized deployment
- ✅ **Auto-Update** — Fetches latest release from GitHub, restarts on update
  - Override with `--no-update` flag

**Configuration:**
```toml
[General]
Port = 18860
MaxPlayers = 20
MaxCarsPerPlayer = 3

[Network]
TickRate = 20
UdpBufferSize = 65535
TcpKeepAliveSec = 15

[Updates]
AutoUpdate = true
```

### Client — Stability

- ✅ **Heartbeat Protocol** — 20-second ping interval with 30-second timeout prevents stale connections
- ✅ **Interpolation** — Smooth vehicle movement via lerp (position) and slerp (rotation)
- ✅ **Error Recovery** — Automatic reconnection with exponential backoff

## Deployment Guide

### Docker

```bash
# Build image
docker build -t highbeam-server:0.5.0 -f server/Dockerfile .

# Run with custom config
docker run -d \
  -p 18860:18860/tcp \
  -p 18860:18860/udp \
  -p 18861:18861/tcp \
  -v $(pwd)/ServerConfig.toml:/app/ServerConfig.toml \
  -v $(pwd)/server.log:/app/server.log \
  highbeam-server:0.5.0 \
  /app/highbeam-server --headless
```

### Systemd (Linux)

```bash
# Install binary to /usr/local/bin/
sudo cp target/release/highbeam-server /usr/local/bin/

# Copy and enable service
sudo cp server/highbeam-server.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable highbeam-server
sudo systemctl start highbeam-server

# Monitor logs
sudo journalctl -u highbeam-server -f
```

### Manual (Linux/macOS/Windows)

```bash
# Build
cd server
cargo build --release

# Run
./target/release/highbeam-server --config ServerConfig.toml
```

## Known Limitations

1. **No GUI** — Administration is command-line only (GUI planned for v0.6.0)
2. **No State Persistence** — Vehicle positions lost on restart (planned for v0.7.0)
3. **No Binary Protocol** — Still using JSON (reduces bandwidth optimization for v0.7.0)
4. **Auto-certificate Generation** — `AutoGenerate=true` not yet implemented; use external tool

## Migration from v0.4.x

**No breaking changes!** v0.5.0 is fully backward compatible with v0.4.3 configs.

Simply:
1. Build `v0.5.0` binary
2. Keep existing `ServerConfig.toml` (or update new options as desired)
3. Run `highbeam-server --config ServerConfig.toml`

**Optional improvements from v0.4.3:**
- Add `[Logging] RotationMaxSizeMb` and `MetricsIntervalSec` options
- Add `[TLS]` section if you want TLS support
- Add `[Updates] AutoUpdate = false` if you prefer manual updates
